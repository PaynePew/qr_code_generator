# Issue Tracker

**Type:** GitHub Issues
**Repo:** PaynePew/qr_code_generator
**CLI:** `gh` (GitHub CLI)

## How skills interact with issues

- **Create:** `gh issue create --title "..." --body "..."`
- **List:** `gh issue list`
- **View:** `gh issue view <number>`
- **Edit:** `gh issue edit <number> --add-label "..."`
- **Close:** `gh issue close <number>`

## Conventions

- Issues should have a clear title and a body describing the problem or feature
- Labels track triage state (see `triage-labels.md`)
- Milestones are optional but useful for grouping related issues
